Archetype

Archetype

Archetype is a 3D first-person shooter game written in C# with Mogre 3D, (a .NET wrapper for Ogre 3D).

Gameplay

Unlike any regular first-person shooters like Counter Strike, users can use superpowers. The goal of this game is to eliminate your enemies before the timer runs out.

Features

  • AI for computer players
  • Created a primitive shape system for accurate collision detection
  • Utilized the Mogre library for skeletal animation and particle systems

Tales of Etherea

Tales of Etherea

Tales of Etherea is a third-person 3D action oriented adventure game. It is my first attempt on large 3D game development using Microsoft XNA Framework. Even though creation of the game is still in progress, some of the major functionality has already been implemented, and is presented in the screenshots above. I have also written a storyline for the game in order to make this game more attractive to players.

Gameplay

The game revolves around the main character of the story. Player can wield any close ranged weapons, such as swords and scythes, to attack enemies while casting powerful magic spells to inflict serious damage on enemies.

Features

  • Custom shader and post-rendering process, such as glowing
  • Used both custom-made particle engine and Dynamic Particle System Framework (DPSF) to produce various particle effects
  • Custom-made animation engine to animate character parts
  • Supported both indoor/outdoor environment
  • Implemented path finding algorithm using Dijkstra's Algorithm
  • Customized model content pipeline objects to precompute bounding boxes for each model

Credits

Lots of models and audio files from the open source community are used. The full credit list will be added once the game is released.

System Requirements

  • Windows operating system

The rest of the system requirements will be listed once the game is released.

Cryptative

Cryptative

Cryptative is a program that combines both encryption and steganography (hide data in another medium) to securely hide files.

Three Encryption Layers

  • AES 128/256-bit encryption
  • Byte reordering by using bytes from another file for positioning
  • Feed the data into another encryption program that you write through its input stream, and get the encryption result from its output stream

Steganography Options

  • Hide encrypted data in files with an end-of-file (EOF) marker
  • Use the low bits of each color per pixel to hide encrypted data
  • Use the low bits of each audio frame in an audio file to hide encrypted data

System Requirements

  • Java Runtime Environment (JRE) 1.7 or above

Unfortunately, this program cannot be shared due to Canada's cryptography export regulations.

JSketch

JSketch is an assignment for CS 349 - User Interfaces, and its purpose is to create simple animation on a canvas. The goal of this assignment is to demonstrate the idea behind Model-View-Controller (MVC).

Features

  • Draw, select, erase, and transform strokes on canvas
  • Export animation an HTML 5 canvas animation without the usage of any third party libraries (here is a sample of an exported animation)
  • Regular features like any other editing software, such as loading, saving, undo, and redo

AeroInvasion

AeroInvasion

Aero Invasion is a two-dimensional aircraft shooting game written in Java. It was originally a high school programming project coded by Vincent Tang and I. I updated a lot of features to the game for version 1.1 in order to enter the ShowCase Competition of University of Toronto.

Gameplay

The player is allowed to rotate freely, and is allowed choose from a list of weapons. Once the game starts, different enemies enter from all directions. The objective of this game is to defeat all hostile aircrafts, and to minimize damage received as much as possible.

Features

  • Can be played with a wiimote, a keyboard with a mouse, or a smartboard/touchscreen.
  • Self-made Star Wars opening crawl
  • LAN network versus mode

  • Michael Laforest's WiiUseJ library

  • Java Runtime Environment (JRE)
  • (Recommended) Rage Italic Font
  • (Optional) Wii remote controller and Bluetooth receiver

MIPS Debugger

A MIPS simulation tool written in Java for debugging purposes. The syntax of the MIPS assembly program is a simplified version of MIPS used in CS241 - Foundations of Sequential Programs.

Feature

  • Execute MIPS assembly program
  • Monitor changes in registers and memory
  • Detect syntax errors

System Requirements
